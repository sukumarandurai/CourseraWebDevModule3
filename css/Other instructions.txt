To start the browser sync

Command:    browser-sync start --server --directory --files "*"

start command to start the browser sync
--server command to use the browser sync as server which in turn serves our files
--directory command to launch the directory structure of the present working directory
--files 
"*" tells if any changes occur in any of the files in the pwd, then reflect in the browser-sync server (browser)

For our web application, the above command is not sufficient bcoz it tells to look for any changes in the files kept in PWD location
What if we had some folder in the PWD and a file inside of it and we are making some changes to that file? It wont reflect in the browser
So, we need to change the above command to have **/* instead of * which means we are telling the browser to look into "any sub folder within this folder(**) and any files within those sub folders(**/*) - as chaikin says - Watch out every directory all the way down in the PWD folder structure"

Command:    browser-sync start --server --directory --files "**/*"

The class 'container-fluid' stretches to the full width of the browser window (minus some margins) and automatically adjusts its width as you change the width of the browser.

The 'container' class has responsive pre-defined widths, so it only adjusts its width at some pre-defined breakpoints.

Whenever we start with a navigation we want that header color stretch till end of the screen no matter what, the size of the container.
